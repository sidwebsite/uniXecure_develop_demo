從根本做起 打造零信任的安全模型

2024 / 01 / 17

在網路安全的演進中，零信任安全模型備受矚目。2010 年， Forrester Research 分析師 John Kindervag 提出了零信任安全模型，主張：「Never Trust, Always Verify 永不信任，一律驗證！」這一原則強調了對企業內外網路進行框架規劃和嚴格存取限制的必要性。隨著時間的推移，全球政府和企業紛紛跟進，不斷推動零信任安全的概念。

台灣政府自 2022 年起，積極規劃零信任架構網路安全戰略，核心機制包括「身分鑑別」、「設備鑑別」和「信任推斷」。金融監督管理委員會（金管會）更在「金融資安行動方案 2.0」中提出逐步導入零信任網路的三大核心機制，以及網路及資源的細化權限管控。2023 年時，因應資安趨勢定義了新方向：資安長聯盟化、生態圈資安和零信任，以零信任架構重新檢視多項金融資安議題，修訂相關資通系統安全自律規範，包括：內外部網路的資源存取、網段隔離、邊界防護。這其中，Forrester New Wave 和 Gartner 的報告提供了實踐零信任安全的建議，並強調了一個關鍵要素：Micro-Segmentation（微分隔）。

Forrester New Wave：「分隔是有效的零信任架構的關鍵。」

  • 主張：微分隔對於零信任 Private 網路至關重要
  • 原因：Private 網路往往是扁平化，最需要進行網路微分隔，以阻止橫向擴散
  • 痛點：APTs、Log4J、勒索病毒等
  • 功能：流量、資產發現、視覺化、作業系統支援度廣，可依層級進行不同分段

Gartner Report：「網路中的零信任等於微分隔。」

  • 定義：可將安全性策略，落實到同一資料中心內任兩個 Workloads 之間的存取層
  • 技術：可以定義細部分類的網路區域，精細化到單一資產和應用程式
  • 能力：流量可視、資產隔離、政策執行、環境支援

Gartner 對市場的微分隔建議如下：

重點面向 描述
網管&資安 網管與資安兩個團隊在選擇導入微分隔技術時，須協同作業與零信任安全架構保持一致性
目標設立 為初期和長期範圍制定正確目標。 漸進式導入，資源與成功最佳化。
工具選擇 為各種 Workload 提供精細可視化。 支援東西向流量分段，減少攻擊面、防止威脅橫向傳播。
基本能力 依 Application 建立規則：身份、Tag、Label 和特徵值。 從雲端來源、資產倉儲、CMDB 等收集上下文資料。 系統建置後，可無痛延展。
微分隔能力 須提供 Ｗorkload 流量可視化。 可透過 AI/ML 提供策略建議和半自動化部署。
架構支援 微分隔架構：Network Overlay, Host, Built-in and/or API 支援不同 Workload 位置(on-premises, hybrid and IaaS) 與環境種類(容器、虛擬機)。

在選擇零信任微分隔解決方案時，Gartner Report 有下列功能的建議：

重點面向 描述
自動策略識別
  • 政策識別：支援自動策略識別、建立和模擬。
  • 政策部署：將透過人為干預進行控制和驗證。
  • 政策變更：更新或新增策略將根據現有流程進行驗證，以防止任何不合規情況。
建立上下文引導出的策略
  • 規則允許：基於 Workload 屬性和關係，而不是透過 IP 位址。
  • 標籤關聯：能夠將 Workload 與多維標籤或標籤關聯起來以自動化策略部署。
  • 收集數據：收集上下文資料，例如：流程 ID、流程 mapping 和流程管理者。
  • 多重來源：收集處理第三方來源的資料（例如：CMDB、load balancer、防火牆、公有雲標籤和編排平台）。
AI/ML 用於智慧型應用程式變更感知細緻度的管理能力
  • 流量意識：需要有流量意識的敏感度。
  • 智慧感知：使用 AI/ML 演算法不斷發現新的、遷移的和變化的 Workload 以及這些 Workload 之間的通訊模式。
  • 決策建議：能夠推薦流量標記、工作流程分組和適當的策略。
  • 通訊可視：對於通訊的可視化很有用。
  • 全面安全：能夠在各個 Workload 層級注入安全性（如：實體機、虛擬機和容器）。
靈活彈性的管理
  • 集中管理：提供跨混合環境集中管理的能力。
  • 策略跟隨：策略綁定相關的 Workloads，當 Workload 在資料中心和雲端中移動、變更或遷移時，皆不影響。
  • 涵蓋度廣：須足夠廣泛涵蓋異質資料中心。
整合與敏捷
  • 整合性高：支援第三方安全產品的整合（如：企業網路防火牆），包括：防火牆管理工具（如：Tufin）。
  • 敏捷開發：允許使用其他安全產品的 API 檢索或推送資料以增強/利用偵測。

因此，深入了解並實踐 Micro-Segmentation 是實現零信任安全的關鍵。這不僅是一種技術手段，更是從根本打造零信任安全模型的重要步驟。在數位時代，零信任安全模型將是確保企業安全的不二之選！
